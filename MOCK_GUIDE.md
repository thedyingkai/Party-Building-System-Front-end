# Mock 数据演示模式使用指南

## 📋 概述

为了解决中期验收时后端未完成、数据库未就绪的问题，我们已经为项目添加了完整的 **Mock 数据演示系统**。这样您可以：

✅ **独立演示所有前端页面**  
✅ **无需后端服务和数据库**  
✅ **一键切换真实/演示模式**  
✅ **完整的业务流程演示**

---

## 🚀 快速开始

### 1. 启用 Mock 模式

打开文件 `src/utils/mockInterceptor.js`，将 `MOCK_ENABLED` 设置为 `true`：

```javascript
// 第 17 行
export const MOCK_ENABLED = true  // ✅ 演示模式
```

### 2. 启动项目

```bash
npm run serve
# 或
pnpm run serve
```

### 3. 访问系统

打开浏览器访问：`http://localhost:8080`

使用演示账号登录：
- **用户名**: `admin` 或任意用户名
- **密码**: 任意密码（Mock 模式下会自动登录成功）

## 🔄 切换回真实后端

当后端开发完成后，只需一步操作：

打开 `src/utils/mockInterceptor.js`，将 `MOCK_ENABLED` 改为 `false`：

```javascript
export const MOCK_ENABLED = false  // ❌ 关闭 Mock，使用真实后端
```

**所有 API 调用将自动切换到真实后端！**

## 🐛 故障排除

### 问题 1：页面显示空白
**解决方案**：检查控制台是否有报错，确认 Mock 文件已正确导入

### 问题 2：某些接口返回空数据
**解决方案**：查看 `src/utils/mockInterceptor.js` 文件，添加对应接口的 Mock 处理

### 问题 3：想看真实的网络请求日志
**解决方案**：打开浏览器开发者工具 Console，搜索 `[Mock]` 查看所有拦截的请求

### 问题 4：需要修改响应数据格式
**解决方案**：在 `mockInterceptor.js` 中找到对应接口，修改 `successResponse()` 的返回结构

---

## 📝 注意事项

1. ⚠️ **Mock 数据仅在前端内存中**，刷新页面后修改会丢失
2. ⚠️ **文件上传**会返回占位图片 URL
3. ⚠️ **所有数据修改**不会持久化到数据库
4. ⚠️ **登录验证**在 Mock 模式下总是成功
5. ✅ **适合演示**，不适合实际业务使用
